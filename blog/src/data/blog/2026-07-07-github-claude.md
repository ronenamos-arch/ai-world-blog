כל מי שעובד עם Claude Code מגיע במהירות לאותה נקודה: הכלי הבסיסי עובד מצוין, אבל ברגע שרוצים agents מותאמים, skills מיוחדים או hooks לאוטומציה - אין תיעוד רשמי שמכסה הכול, וכל אחד ממציא את הגלגל מחדש בעצמו.

**הפתרון הכי מהיר הוא לא לקרוא מדריכים - הוא להיכנס לפרויקטים מגיטהאב שאנשים אחרים כבר בנו**, ולראות בדיוק איך הם מארגנים agents, commands ו-MCP configs בפרויקטים אמיתיים. הנה עשרה פרויקטים מגיטהאב ששווה להכיר, מסודרים לפי מה שהם מלמדים.

## 1. תשתית מתקדמת מניסיון יומיומי

<a href="https://github.com/affaan-m/everything-claude-code" style="display:block;width:fit-content;text-align:center;padding:0.5rem 1rem;margin:1rem auto;border-radius:0.5rem;background:linear-gradient(135deg,#0d0d0d,#2e2a12);color:#d4c95e;font-weight:700;font-size:0.95rem;text-decoration:none;border:1px solid #8a7f2a;">🔗 everything-claude-code</a>

### מה מקבלים?

• סט שלם של agents, skills, hooks ו-MCP configs שנבנה מתוך יותר מ-10 חודשים של שימוש יומיומי אמיתי

• דוגמאות שהוכיחו את עצמן גם בתחרות hackathon

**למי מתאים:** מי שרוצה לראות איך נראית תשתית Claude Code בשלה, לא רק הדגמה בסיסית.

**למי מתאים:** מי שרוצה להבין את ההבדלים הארכיטקטוניים בין כלי הקידוד השונים.

## 3. סוכנים בתפקידים ארגוניים

<a href="https://github.com/garrytan/gstack" style="display:block;width:fit-content;text-align:center;padding:0.5rem 1rem;margin:1rem auto;border-radius:0.5rem;background:linear-gradient(135deg,#0d0d0d,#2e2a12);color:#d4c95e;font-weight:700;font-size:0.95rem;text-decoration:none;border:1px solid #8a7f2a;">🔗 gstack</a>

**למי מתאים:** מי שרוצה להריץ כמה "תפקידים" של סוכנים במקביל על אותו פרויקט.
